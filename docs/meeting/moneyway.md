# Moneyway 진행사항 및 논의사항 정리

## 1. 현재까지의 진행사항 요약

### 1. 구글 로그인 기능

- Google ID Token 기반 회원가입/로그인 API를 구현
- 로그인 성공 시 access token과 refresh token을 발급하는 구조로 구현

### 2. 주도 섹터 / 주도 테마 기능

- 섹터/테마 화면에서 사용할 주도 섹터, 주도 테마 API를 구현
- 장중에 수집된 종목 현재가 스냅샷을 기준으로 당일 주도 섹터/테마를 계산
- 특정 섹터/테마 안에서 종목들을 등락률 순으로 조회할 수 있는 API도 구현

### 3. 일자별 주도 섹터 / 주도 테마 기능

- 특정 날짜를 지정해 해당일의 주도 섹터/테마를 조회할 수 있도록 구현
- 해당 날짜의 장중 스냅샷이 있으면 장중 스냅샷을 우선 사용
- 장중 스냅샷이 없으면 종목별 일봉 데이터를 사용해 주도 섹터/테마를 계산
- 주말이나 휴장일처럼 해당 날짜의 데이터가 없으면 직전 거래일 데이터를 사용

섹터/테마 데이터 구성

- 섹터 분류 데이터는 토스증권을 크롤링해 구성 : https://www.tossinvest.com/
- 테마 분류 데이터는 주달 데이터를 크롤링해 구성 : https://www.judal.co.kr/

주도 섹터/테마 계산 기준

- 각 종목의 당일 누적 거래대금과 전일 대비 등락률을 사용
- 섹터/테마별로 소속 종목들의 거래대금을 합산
- 거래대금이 큰 종목의 등락률이 더 크게 반영되도록 거래대금 가중 등락률을 계산
- 상승 주도는 상승 종목 비율과 상승 종목의 거래대금 비중을 함께 반영
- 하락 주도는 하락 종목 비율과 하락 종목의 거래대금 비중을 함께 반영
- 특정 1개 종목에 거래대금이 과도하게 쏠린 경우 점수를 낮추는 보정값을 적용
- 거래대금 1000억 원 이상인 섹터/테마만 주도 후보로 사용
- 상승 주도는 상승 종목이 최소 3개 이상, 하락 주도는 하락 종목이 최소 3개 이상인 경우만 후보로 사용
- 최종적으로 거래대금 규모, 가중 등락률, 상승/하락 확산도, 거래대금 쏠림 보정을 조합한 점수로 순위를 산정

### 4. 발견 / 탐색 화면 기능

- 거래대금 상위, 거래량 상위, 급상승, 급하락 종목 데이터를 제공하는 API를 구현
- 인기 검색 종목 API를 구현
- KOSPI/KOSDAQ 지수와 상승/하락 종목 수를 제공하는 API를 구현

## 2. 현재 사용 중인 한국투자증권 OpenAPI

현재 프로젝트에서는 한국투자증권 OpenAPI를 시장 데이터 수집과 주도 섹터/테마 계산을 위한 원천 데이터 수집에 사용하고 있음.

| 공식 API명 | Endpoint | 사용하는 기능 |
| --- | --- | --- |
| 접근토큰발급(P) | `POST /oauth2/tokenP` | 한국투자증권 REST API 호출에 필요한 access token 발급 및 갱신 |
| 주식현재가 시세 | `GET /uapi/domestic-stock/v1/quotations/inquire-price` | 주도 섹터/테마용 현재가 스냅샷 수집, 인기 검색 종목 등락률 보강 |
| 거래량순위 | `GET /uapi/domestic-stock/v1/quotations/volume-rank` | 발견/탐색 화면의 거래량 상위 종목 조회 |
| 거래량순위 | `GET /uapi/domestic-stock/v1/quotations/volume-rank` | 발견/탐색 화면의 거래대금 상위 종목 조회. 별도 거래대금순위 API가 아니라 `FID_BLNG_CLS_CODE=3` 옵션으로 거래금액순 조회 |

| HTS조회상위20종목 | `GET /uapi/domestic-stock/v1/ranking/hts-top-view` | 발견/탐색 화면의 인기 검색 종목 코드 조회 |

## 3. 논의사항

### 1. 한국투자증권 OpenAPI 관련 논의사항

- REST API 호출 유량 제한이 있음. API 초기 신청 후 3일 동안은 초당 1회 수준으로 호출이 제한되고, 3일 이후부터는 초당 18회 수준으로 호출 가능
- 1계좌당 하나의 API만 등록할 수 있고, 한 계정에서 총 2개의 API 등록 제한이 있음
- 실시간 WebSocket API는 종목을 구독해서 사용하는 방식이고, 한 번에 구독 가능한 종목 수 제한(41종목)이 있어 전체 종목 실시간 수집에는 적합하지 않음
- 일부 랭킹 API는 통합 시장(`UN`) 조회를 지원하지 않고 KRX(`J`) 또는 NXT(`NX`)처럼 시장 구분을 나눠 호출해야 함.

현재 호출 제한에 대응하는 방식

- 거래대금 상위, 거래량 상위, 급상승, 급하락, 인기 검색, 지수 데이터는 worker가 API 호출 사이에 간격을 두고 수집한 뒤 1분 단위로 데이터베이스에 저장
- 프론트는 한국투자증권 API가 아니라 데이터베이스에 저장된 최신 데이터를 조회
- 주도 섹터/테마 계산용 현재가는 전체 종목을 대상으로 한 번에 호출하지 않고, 1초당 1종목 수준으로 순차 호출해 유량 제한을 피하는 방식으로 수집
- 주도 섹터/테마 화면도 프론트 요청 시 한국투자증권 API를 직접 호출하지 않고, 미리 저장된 장중 데이터 또는 일봉 데이터를 기준으로 계산
- 향후 주식 현재가 실시간 데이터가 필요할 경우 코스콤 등 유료 실시간 데이터 공급원 사용을 고려 (https://www.koscom.co.kr/portal/main/contents.do?menuNo=200653)

### 2. 섹터/테마 분류 데이터 관련 논의사항

- 섹터 분류는 토스증권 데이터를 기반으로 구성했기 때문에 카테고리 범위가 비교적 넓음
- 예를 들어 반도체 섹터 안에 반도체 전공정, 후공정, 장비, 소재 등 다양한 성격의 종목이 하나의 반도체 섹터로 묶일 수 있음
- 이 경우 실제로는 특정 하위 산업군만 강세인데, 화면에서는 전체 반도체 섹터가 주도 섹터처럼 보일 수 있음
- 테마 분류는 주달 데이터를 기반으로 구성했기 때문에 토스 섹터보다 세분화된 테마 구성이 가능함
- 다만 주달 테마에 속하지 않는 종목도 많아, 모든 종목이 테마 계산에 반영되지는 않음
- 결과적으로 섹터는 범위가 넓어 해석이 거칠 수 있고, 테마는 세분화되어 있지만 커버리지 공백이 있을 수 있음
- 현재 DB 기준 ETF/ETN을 제외한 계산 대상 2,877종목 중 섹터 미분류 종목은 416개, 테마 미분류 종목은 326개 존재
- 한 종목이 여러 섹터/테마에 동시에 속하는 경우가 있어, 특정 종목의 상승이 여러 섹터/테마 점수에 중복 반영될 수 있음
- 특히 테마는 다중 소속 종목이 많아 특정 대형주 또는 인기 종목의 움직임이 여러 테마를 동시에 끌어올리는 효과가 발생할 수 있음
- 카테고리별 종목 수 편차가 큼. 섹터는 최대 183종목, 테마는 최대 446종목까지 포함되는 카테고리가 있어 작은 카테고리와 큰 카테고리의 점수를 단순 비교하기 어려움
- 테마 데이터에는 `테마없음`, `지방코스닥기업`, `MSCI Korea`, `밸류업`처럼 산업/사업 테마라기보다 분류 태그에 가까운 항목도 포함될 수 있음
- 현재는 이런 비사업성 테마를 별도로 제외하는 필터가 없어, 주도 테마로 노출되기 전에 제외 기준 또는 별도 분류 기준을 추가할 필요가 있음

### 3. 주도 섹터/테마 계산식 관련 논의사항

현재 주도 섹터/테마는 단순 거래대금 순위가 아니라, 거래대금이 실제로 상승 또는 하락 방향성과 함께 움직였는지를 점수화하는 방식으로 계산

기본 계산 값

- `trade_amount`: 섹터/테마에 속한 종목들의 당일 누적 거래대금 합계
- `weighted_change_rate`: 거래대금이 큰 종목의 등락률이 더 크게 반영된 가중 등락률
- `advance_ratio`: 섹터/테마 내 상승 종목 비율
- `up_trade_amount_ratio`: 상승 종목들이 차지하는 거래대금 비중
- `decline_ratio`: 섹터/테마 내 하락 종목 비율
- `down_trade_amount_ratio`: 하락 종목들이 차지하는 거래대금 비중
- `concentration_penalty`: 특정 1개 종목에 거래대금이 과도하게 쏠린 경우 점수를 낮추는 보정값

상승 주도 점수

```text
score =
  ln(1 + trade_amount)
  * max(weighted_change_rate, 0)
  * up_trade_amount_ratio
  * min(1, advance_ratio / 0.6)
  * concentration_penalty
```

하락 주도 점수

```text
score =
  ln(1 + trade_amount)
  * abs(min(weighted_change_rate, 0))
  * down_trade_amount_ratio
  * min(1, decline_ratio / 0.6)
  * concentration_penalty
```

계산식에서 논의가 필요한 부분

- 거래대금이 큰 섹터/테마가 점수에서 유리해질 수 있음
- 대형주 1~2개가 강하게 움직이면 관련 섹터/테마 점수가 과도하게 높아질 수 있음
- 이를 줄이기 위해 현재는 1위 종목 거래대금 쏠림 보정과 상승/하락 종목 최소 3개 조건을 적용하고 있음
- 다만 종목 수가 적은 테마는 최소 3개 조건 때문에 제외될 수 있고, 종목 수가 많은 섹터는 확산도 계산에서 상대적으로 유리하거나 불리할 수 있음
- 향후에는 카테고리별 종목 수, 시가총액, 수급, 최근 며칠간의 지속성 등을 추가로 반영할지 논의가 필요함
